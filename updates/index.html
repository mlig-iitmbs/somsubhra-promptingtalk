    Hands-on Activity!!¶

No additional materials are required; if you’ve followed along with the sessions, you’ll have everything you need to complete it.

    Motivation & Objective¶

    In traditional exams, teachers manually evaluate handwritten answer sheets - a time-consuming process often prone to human error. In this task, you will use Google’s Gemini LLM to build an AI system that can automatically read, understand and evaluate student’s handwritten answers from an image. To achieve a good performance, the LLM must handle challenges like inconsistent spelling, incomplete words, differences in handwriting styles, effectively distinguish actual answers from extra markings such as scribbles or underlines, which adds to the complexity.

    Example¶


    Let’s look at this sample image. A answer sheet of a student consisting of SN (serial number), Statement (the questions) and the answer column consisting of the student’s handwritten answer - True/False, for each of the \(10\) problems.

    You have the pre-defined answers and now you can compute the student’s total score by matching each of the responses with the correct key. For eg., in this case, we have \(3\) matches (Questions \(1\), \(4\) and \(7\)), hence a score of \(\frac{3}{10}\). Here’s a sequence diagram version of the flow:

    sequenceDiagram
    participant User
    participant AI_System
    participant GeminiLLM
    participant EvaluationLogic

    User->>AI_System: Submit answer sheet image + prompt
    AI_System->>GeminiLLM: Send image and evaluation prompt
    GeminiLLM->>AI_System: Return extracted answers (True/False)
    AI_System->>EvaluationLogic: Compare answers with key
    EvaluationLogic->>EvaluationLogic: Initialize score = 0, total = 10

    loop For each question 1..10
        EvaluationLogic->>EvaluationLogic: Compare extracted answer[i] with key[i]
        alt Correct answer
            EvaluationLogic->>EvaluationLogic: score = score + 1
        else Incorrect answer
            EvaluationLogic->>EvaluationLogic: score unchanged
        end
    end

    EvaluationLogic->>AI_System: Return final score = score / total
    AI_System->>User: Display total score

    🎯 Milestones¶

    • Create a Gradio web app running in Colab (Gradio provides user-friendly interface — no need for custom HTML, CSS or server setup… just a few lines of Python code and you’re done!) that:

      • accepts images (Download here –> sheet1, sheet2, sheet3, sheet4) of handwritten answer sheets in a fixed layout.
      • uses Gemini 2.5-flash’s (same model we used in the 1st session) multimodal capabilities to interpret the handwritten text.
      • compares each extracted answer with the set of predefined correct answers.
      • computes and displays the question number, student’s transcribed response, correct key, the result (Correct/Incorrect) and the total score (refer to the demo interface images).
      • (Optional: you can further make the flow dynamic by adding some more lines of Python code to save the scores from the previous sheet uploads, thus having a proper record.)

    📝 Evaluation¶

    Pointers for better performance?¶

    • LLM’s Accuracy: How reliably the model can read the student’s handwriting (which can be messy, slanted, or oddly capitalized, check the images for ref.) - Depends on the prompt

    • Scoring logic: ensures the transcribed answer is correctly mapped to the predefined correct key and the score is accurately computed.

    Important Notes¶

    Disclaimer: You are provided with only 4 samples1 to test on, hence the nuance/actual complexity of such image processing tasks cannot be understood from this activity. This is meant to provide you a hands-on opportunity on a real-word problem.

    • Do NOT use any separate OCR tools (for eg, Tesseract). Instead, rely solely on prompt engineering techniques by crafting and refining prompts to help the AI model accurately extract and interpret handwritten answers from the provided images.

    • The entire solution must be fully implemented within a single Google Colab notebook. Nothing fancy! (No need of separate webapp archs e.g., no Flask, Streamlit, or external backends etc.)

    • You can use the same free tier API key that you had created initially. It should be more than enough, given the less number of samples & the total call limits per day for this model. Goes without saying, accelerators/ GPUs not needed, utilise the standard CPU runtime of Colab.

    • You are allowed to refer to AI tools (like ChatGPT, Perplexity, etc.) for solving this task, but please use them responsibly! Don’t rely on them to generate the entire solution. Instead, take help when you’re stuck and use them to guide you through each milestone step by step. Please declare the use of AI assistance in your submission (possibly in a readme.md file).

    ✅ Submission Guidelines¶

    • Execute the Colab notebook. Download (export) it in .ipynb format (NOT .py) with the cell outputs visible.

      • Ensure you capture contains: the transcribed answers for each question, comparison table (student’s answer vs. correct key) and total score summary.

    • Finally, zip the .ipynb, images & readme file - submit this single rollnum.zip file in the GForm.
